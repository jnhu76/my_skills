# My Skills

个人 Skill Pack 收藏库，主要服务于以下场景：

- 通用问题分析
- 计算机系统分析
- Debug / 故障定位
- Agent 工作流排查
- 架构与过程诊断
- 需要多轮追问、状态保存、上下文收缩的问题

每个技能包独立成目录，可单独使用，也可配合辅助工具使用。

## 当前收录技能

### Phenomenon Skill — 现象驱动分析技能

核心理念：**先扩观察面，再缩分析窗。**

不把用户输入的"结论句"直接当结论，而是通过分阶段分析，逐步逼近问题的真实结构。

分析流程分为 6 个阶段：

1. **现象展开** — 把"结论句"翻译成"可观察现象"，区分事实、解释、情绪
2. **矛盾整理** — 找出互相冲突却同时存在的现象，矛盾是分析的入口
3. **分层拆解** — 从历史/结构/制度/文化/技术/环境多维度展开
4. **机制假设** — 构造 1~3 个"当前最强解释"，保持竞争性
5. **边界确认** — 明确结论在什么条件下成立、什么条件下不成立
6. **有限结论** — 给出分层、有限、可验证的结果，不装全知

更详细的说明见 [phenomenon_skill/README.md](./phenomenon_skill/README.md)。

## 结构规范

每个 Skill Pack 至少包含：

```
skill_name/
├── README.md           # 必需
├── mainfest.json       # 必需，元数据与方法论
├── general/
│   └── skill.md        # 至少一个
├── python/             # 可选
├── shared/templates/   # 可选
└── examples/           # 可选
```

新增 Skill 的详细规范见 [CONTRIBUTING.md](./CONTRIBUTING.md)。
